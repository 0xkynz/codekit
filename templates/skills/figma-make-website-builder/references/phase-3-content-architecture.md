# Phase 3 — Content Architecture

**Role:** Act as a Conversion Copywriter.

## Purpose

For each page defined in Phase 1, deliver complete website copy optimized for conversion. Output populates Figma Make's content layers.

## Deliverables Per Page

### 1. Hero Section

| Element | Spec | Example |
|---------|------|---------|
| Headline | ~6 words, power word, specific benefit | "Ship Products That Users Love" |
| Subheadline | ~15 words, expand on headline, address pain | "Stop guessing what customers want. Our platform turns user feedback into product decisions in minutes." |
| Primary CTA | 2-4 words, action-oriented, urgency | "Start Free Trial" |
| Secondary CTA | 2-4 words, lower commitment | "Watch Demo" |
| Supporting element | Social proof micro-stat | "Trusted by 10,000+ teams" |

**Character count targets:**
- Headline: 25-40 characters
- Subheadline: 80-120 characters
- CTA: 10-20 characters

### 2. Feature Sections (3 blocks minimum)

For each feature block:

```
Feature Block:
  Icon/Illustration: [describe visual]
  Headline (H3): 4-6 words — benefit-focused, not feature-focused
  Description: 2-3 sentences, 40-60 words
  Optional CTA: "Learn more →"
```

**Framework — translate features into benefits:**
- Feature: "Real-time analytics dashboard"
- Benefit: "See What's Working, Instantly"
- Description: "Track every metric that matters in real-time. Our dashboard surfaces insights the moment they happen, so you can double down on what works and fix what doesn't — before it costs you."

### 3. Social Proof Section

**Testimonial framework:**
```
Testimonial Card:
  Quote: 2-3 sentences (40-60 words)
  Attribution:
    Name: Full name
    Title: Job title, Company
    Avatar: Headshot placeholder
  Rating: 5 stars (optional)
```

**Stats section:**
```
Stats Bar:
  Stat 1: "10,000+" | "Active Users"
  Stat 2: "99.9%" | "Uptime"
  Stat 3: "4.9/5" | "Customer Rating"
  Stat 4: "150+" | "Countries"
```

**Logo bar:**
List 6-8 recognizable company names or placeholders.

### 4. FAQ Section (8 questions)

Structure each FAQ with:
```
Q: [Question — use customer language, not jargon]
A: [Answer — 2-4 sentences, direct, address concern behind the question]
```

**FAQ categories to cover:**
1. Product/service scope — "What does X do?"
2. Pricing/plans — "How much does it cost?"
3. Getting started — "How do I get started?"
4. Technical — "Does it work with [X]?"
5. Data/security — "Is my data safe?"
6. Support — "What if I need help?"
7. Comparison — "How is this different from [competitor]?"
8. Commitment — "Can I cancel anytime?"

### 5. Footer Content

```
Footer:
  Column 1 — Product:
    Features, Pricing, Integrations, Changelog, Roadmap
  Column 2 — Company:
    About, Blog, Careers, Press, Contact
  Column 3 — Resources:
    Documentation, Help Center, Community, API Reference
  Column 4 — Legal:
    Privacy Policy, Terms of Service, Cookie Policy

  Social Links: Twitter/X, LinkedIn, GitHub, Discord
  Newsletter: Email input + "Subscribe" CTA
  Copyright: "© 2025 [Company]. All rights reserved."
```

## Emotional Triggers

Apply these conversion psychology principles throughout:

| Trigger | Application |
|---------|-------------|
| **Urgency** | "Start your free trial — no credit card required" |
| **Scarcity** | "Limited to 500 beta users" |
| **Authority** | "Trusted by Fortune 500 companies" |
| **Social proof** | "Join 10,000+ teams already using X" |
| **Reciprocity** | "Free forever plan available" |
| **Loss aversion** | "Don't let another sprint go to waste" |

## Power Words

Use these throughout headlines and CTAs:
- Action: Launch, Build, Ship, Scale, Unlock, Discover, Transform
- Benefit: Effortless, Instant, Powerful, Seamless, Simple
- Urgency: Now, Today, Limited, Exclusive, Free
- Trust: Proven, Secure, Guaranteed, Reliable, Trusted

## Heading Hierarchy

| Tag | Use | Example |
|-----|-----|---------|
| H1 | One per page, hero headline | "Ship Products That Users Love" |
| H2 | Section headings | "Why Teams Choose Us" |
| H3 | Feature/card titles | "Real-Time Analytics" |
| H4 | Subsection or accordion headers | "How does billing work?" |
| Body | Descriptions, paragraphs | Regular content |
| Small | Labels, captions | "No credit card required" |
